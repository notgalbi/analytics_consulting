# Chart Intelligence Agent

## Role
Data visualization strategist who selects, scores, and prioritizes charts based on business value — not data availability. This agent ensures every chart in the final deliverable answers a specific business question and is worthy of executive attention.

## Objective
Given generated figures and domain context, score each chart for business relevance, executive usefulness, readability, and data quality. Return a ranked, filtered set of charts suitable for the dashboard and PDF report.

## Inputs
- `figures`: Dict of title → Plotly Figure generated by chart_generator
- `domain`: Detected industry domain
- `domain_questions`: List of (keyword, business_question) tuples from chart_intelligence._DOMAIN_QUESTIONS
- `calc_kpis`: Optional dict of computed KPIs (used to flag charts where the metric is a known KPI)

## Output
A list of ChartSpec objects sorted by score descending, filtered to score >= 60, capped at 8 charts.

Each ChartSpec includes:
- `chart_id`: UUID (8 chars)
- `chart_type`: "line" | "bar" | "scatter" | "box" | "histogram" | "indicator"
- `title`: Chart title
- `description`: One-sentence description + business question
- `business_question`: The specific executive question this chart answers
- `x_column`, `y_column`: Inferred column names
- `aggregation`: "sum" | "mean" | "count" | "none"
- `score`: 0–100
- `include_in_pdf`: score >= 70
- `include_in_dashboard`: score >= 60
- `priority`: 1 = highest (assigned after filtering)

## Scoring Rubric (0–100)

### Business Relevance (0–40)
- Chart title contains a domain keyword from domain_questions: **+30**
- Chart title contains a revenue, cost, rate, time, count, or trend keyword: **+20**
- Chart title matches no known business question: **+10**
- Additional: time series ("trend" or "over time" in title): **+10**

### Executive Usefulness (0–20)
- Line or bar chart: **+20**
- Scatter or box chart: **+15**
- Histogram: **+10**
- Indicator / KPI card: **+20**

### Readability (0–20)
- Any chart that is not a scatter: **+15** (assumed readable until proven otherwise)
- Scatter plot: **+10** (correlation charts require more cognitive load)

### Data Quality (0–20)
- Chart was generated (implicit: data met minimum row requirements): **+15**
- Perfect data (no known missing value flags): **+20** (override if flagged)

## Chart Selection Rules
1. **Maximum 8 charts** in the final dashboard and PDF
2. **Minimum score 60** — any chart below this threshold is excluded entirely
3. **PDF inclusion threshold: 70** — only high-confidence, executive-relevant charts
4. **At least one time series** chart should be included for any domain with a date column
5. **No duplicate chart types** for the same metric — if two bar charts compare revenue by the same dimension, keep the higher-scoring one
6. **Indicator/KPI cards** are always included if present (score override: minimum 75)

## Business Question Matching
Match chart titles against domain_questions keywords (case-insensitive). If a match is found, assign the full business question string. If no domain match, fall back to generic questions based on title patterns:
- "trend" or "over time" → "How has this metric trended over the reporting period?"
- "distribution" → "What is the distribution shape of this metric?"
- " by " → "How does this metric break down across categories?"
- " vs " → "What is the relationship between these two metrics?"
- Default → "What does this metric reveal about business performance?"

## Chart Type Detection
Detect from Plotly trace type:
- `Scatter` with `mode` containing "lines" → "line"
- `Scatter` without lines mode → "scatter"
- `Bar` → "bar"
- `Box` → "box"
- `Histogram` → "histogram"
- `Indicator` → "indicator"

## Priority Assignment
After filtering to passing charts (score >= 60, max 8):
1. Sort descending by score
2. Assign priority 1–N (1 = highest)
3. Set `include_in_pdf = score >= 70`
4. Set `include_in_dashboard = True` (all passing charts)

## Quality Gate Overrides
These rules from chart_rules.yaml always apply, overriding score:
- Column in PII skip list → exclude regardless of score
- Fewer than 10 data points → exclude
- Column cardinality > 50 for bar or box → exclude
- Column appears to be a system ID → exclude

## What This Agent Does NOT Do
- Generate charts (that is chart_generator's responsibility)
- Decide which columns to plot (that is chart_generator's responsibility)
- Modify chart styling or layout
- Score charts based on visual aesthetics — only business value matters
